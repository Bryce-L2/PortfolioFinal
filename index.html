01

Market Gene

Investors often struggle to narrow down stock picks or validate trade setups without a structured framework. Market Gene solves this by asking them a couple of questions, then returning stock recommendations. A separate futures module applies ICT confluence analysis to ES futures setups, scoring trade strength and projecting potential profit per contract.

Java ICT Concepts
Design Thinking

I structured the program around two completely different market workflows: stocks and futures, both accessible from a persistent main menu loop. The futures module accumulates confluence points across four conditions and maps the total to a risk tier, giving the user a meaningful, graduated output rather than a simple buy or sell.

Challenge Overcome

Handling invalid user inputs without crashing the program was the core challenge. I added input validation at every prompt and a default case in each switch statement.

Stock Market Simulator

New investors have no way to practice trading before committing real capital. This paper trading simulator gives users $10,000 in virtual cash and a full trading interface. Search any ticker, and execute buys and sells at real 15 minute delayed prices pulled live from the Alpha Vantage API. A portfolio tracker displays current holdings, live value, and profit/loss in real time. Real chart data coming soon.

Java Java Swing Maven Alpha Vantage API
Design Thinking

I modeled the simulator around three core data structures: a cash balance, a holdings map, and a transaction history. Every trade routes through dedicated methods that update all three atomically, mirroring how a real brokerage account tracks positions.

Challenge Overcome

The Alpha Vantage free tier allows only 25 API calls per day, and each stock search consumes two calls, one for the current price and one for the chart history. I had to carefully design the request flow to cache responses where possible and surface a clear error state when the daily limit was hit, so the app never silently fails.

Bullish or Bearish AI

Reading candlestick charts to call the next move takes years to learn, and everyone reads them a little differently. This project trains a CNN to do it instead. It looks at a 50-candle chart image of ES or NQ futures and predicts whether the next day will be bullish or bearish. Built in Python with PyTorch across three scripts, and trained to about 91% accuracy. I also tested it on a real TradingView screenshot it had never seen and it called it right.

Python PyTorch CNN NumPy Pillow
Design Thinking

I split the system into three scripts so I could test each part on its own: data.py makes and labels the candlestick chart images, train.py builds and trains the CNN on that dataset, and predict.py loads the trained model to read a new chart. I treated the chart as an image instead of a list of numbers, so the model learns the visual patterns the same way I do when I look at a chart, rather than me hand-coding a bunch of indicators.

Challenge Overcome

My first model looked accurate in training but fell apart on real charts. It had learned the quirks of how I was generating the images instead of the actual price action. I regenerated the whole dataset with cleaner, more consistent charts and retrained, which got it to around 91% and made it actually work on a real TradingView screenshot.

04

OBD-II RPM LED Visualizer

Factory dashboards give drivers almost no feedback on engine behavior. This project reads live RPM data from a vehicle's OBD-II port via a Freematics UART adapter and maps it onto a NeoPixel strip with bouncy physics. The LEDs climb purple, fade to pink, then white as RPM rises, and the full strip strobes white at redline. Built and tested on a 2017 Ford Fusion Energi, but compatible with any OBD-II vehicle (1996+).

C++ / Arduino OBD-II UART (Freematics) Adafruit NeoPixel Hardware Serial OBD-II / PID_RPM Arduino Uno R3
Design Thinking

I designed the system in three layers: communication using the freematics to talk to the Arduino to get real data from the car's ECU, physics, a bounce algorithm that rises fast and falls naturally for a realistic feel, and output using NeoPixel color gradient mapped across the RPM range with a redline strobe effect. Separating the layers made it easy to test and tune each one independently.

Challenge Overcome

The Freematics adapter uses hardware serial on pins D0/D1, which conflict with the Arduino's USB upload port. Every sketch upload requires unplugging D0 and D1, uploading, then reconnecting. I also traced an early wiring issue to the VCC pin: the adapter is powered by the OBD-II port directly, and connecting its VCC to the Arduino's 5V rail caused conflicts that corrupted serial communication.
